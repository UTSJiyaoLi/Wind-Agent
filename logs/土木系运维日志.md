# 土木系运维日志

## 2026-04-14 14:32:09 +08:00 安全重启检查命令序列（不打断 tmux/GPU/vLLM）

### 0) 本地准备（只读）
`powershell
Set-Location C:\wind-agent
`

### 1) 远端只读巡检（不改任何进程）
`powershell
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "tmux ls"
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "nvidia-smi --query-compute-apps=gpu_uuid,pid,process_name,used_gpu_memory --format=csv,noheader"
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "ps -ef | grep -E 'vllm|api_server|openai.*server' | grep -v grep"
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "curl -sS --max-time 5 http://127.0.0.1:8001/v1/models"
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "curl -sS --max-time 5 http://127.0.0.1:8787/health"
`

### 2) 判定规则（必须遵守）
- 8001 vLLM 正常：禁止重复启动 vLLM。
- 8787 健康正常：禁止重启现有后端。
- 任一现有 tmux 会话：禁止 kill/stop/attach 后改写。
- 发现 GPU 已被占用：禁止抢占或切换 CUDA_VISIBLE_DEVICES。

### 3) 仅当 8787 不健康时：并行拉起新后端（蓝绿，不打断旧会话）
`powershell
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "tmux new-session -d -s wind-rag-api-bg 'cd /share/home/lijiyao/CCCC/Wind-Agent && source ~/.bashrc && conda activate rag_task && python scripts/search/rag_local_api.py --host 127.0.0.1 --port 8788 --llm-base-url http://127.0.0.1:8001 --llm-model \'"
ssh -J lijiyao@172.30.3.166 lijiyao@gpu6000 "curl -sS --max-time 8 http://127.0.0.1:8788/health"
`

### 4) 本地只切隧道/URL，不动远端旧服务
`powershell
# 新开隧道到 8788（保留原 8787）
ssh -N -L 8788:127.0.0.1:8788 -J lijiyao@172.30.3.166 lijiyao@gpu6000
# 前端 Backend URL 临时改为 http://127.0.0.1:8788
`

### 5) 回滚（无侵入）
- 前端 URL 改回 8787。
- 仅在确认新会话无用后，手动结束 wind-rag-api-bg（不触碰其他 tmux）。
